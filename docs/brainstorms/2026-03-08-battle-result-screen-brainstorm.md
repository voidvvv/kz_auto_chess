# Brainstorm: Battle Result Screen (战斗结算界面)

**Date:** 2026-03-08
**Status:** Ready for Planning

---

## What We're Building

在战斗结束后增加一个独立的结算界面，显示战斗统计信息和资源奖励，而不是直接进入准备阶段。

**核心需求：**
- 战斗结束后显示结算界面
- 显示击杀数、存活数、剩余血量、伤害统计
- 根据胜负调整金币奖励
- 用户手动确认后进入下一轮准备阶段

---

## Why This Approach

**当前问题：**
- 战斗结束后直接跳转，缺乏反馈感
- 玩家不清楚战斗表现如何
- 无法查看详细统计数据
- 胜负对金币的影响不明显

**收益：**
- 增强战斗反馈和成就感
- 让策略调整更有依据
- 符合主流自走棋游戏体验（TFT、云顶之弈）

---

## Key Decisions

### 1. 实现方式：独立的 BattleResultScreen

创建新的结算界面类，遵循现有的 Model-Update-Manager-Render 模式。

**原因：**
- 与现有 GameScreen 解耦，职责清晰
- 可独立渲染和输入处理
- 便于未来扩展更多结算功能

### 2. 流程修改

```
PLACEMENT → BATTLE → BATTLE_RESULT (点击确认) → PLACEMENT
```

**影响范围：**
- `GameScreen` 需要新增 `BATTLE_RESULT` 状态
- `BattleUpdater.endBattle()` 需要收集战斗数据后切换到结算界面
- `BattleResultScreen.show()` 接收结算数据并显示

### 3. 金币奖励规则：连胜奖惩

| 结果 | 额外金币 | 备注 |
|------|---------|------|
| 胜利 | +3 | 基础奖励 |
| 失败 | +1 | 安慰奖 |
| 连胜(2+) | 额外+1 | 累积 |
| 连败(2+) | 额外+1 | 补偿 |

**注：** 连胜/连败由现有 `EconomyCalculator` 处理，此处只需在结算界面显示。

### 4. 结算信息内容

| 信息项 | 显示格式 |
|--------|---------|
| 战斗结果 | 🏆 胜利 / 💀 失败 |
| 击杀数 | 击杀: 5/8 |
| 存活单位 | 存活: 3/8 |
| 平均剩余血量 | 平均血量: 45% |
| 总造成伤害 | 造成伤害: 2,450 |
| 总承受伤害 | 承受伤害: 1,890 |
| 本轮金币收入 | +8 金币 (基础5 + 利息1 + 连胜1 + 胜利+3) |

### 5. 连胜/连败显示

在结算界面顶部显示当前连胜/连败 streak：
```
🔥 连胜 3 胜 或 💀 连败 2 局
```

### 6. MVP 单位

显示本轮表现最好的单位：
- 优先级：击杀数 > 造成伤害 > 存活状态
- 显示单位名称、星级、关键数据
```
🌟 本轮MVP: ⭐⭐ 战士队长 (击杀: 3, 伤害: 850)
```

### 7. 敌方信息

显示敌方战斗统计：
```
敌方存活: 2/8  敌方造成伤害: 1,200
```

### 8. 视觉区分

- **胜利**：金色/绿色主题，🏆 图标
- **失败**：红色/灰色主题，💀 图标

## Implementation Notes

### 需要新增的类

```
screens/
  └── BattleResultScreen.java       # 结算界面
model/
  └── BattleResultData.java         # 结算数据模型
  └── BattleStats.java              # 战斗统计数据
```

### 需要修改的类

```
screens/
  └── GameScreen.java               # 添加 BATTLE_RESULT 状态
updater/
  └── BattleUpdater.java            # endBattle() 收集数据并切换界面
logic/
  └── EconomyCalculator.java        # 添加胜负奖励计算
```

### 数据收集点

在战斗过程中需要记录：
- `BattleCharacter.kills` - 单位击杀数（新增字段）
- `BattleCharacter.damageDealt` - 造成伤害总量（新增字段）
- `BattleCharacter.damageTaken` - 承受伤害总量（新增字段）
- `DamageEvent` - 伤害事件（已存在，可用于统计）
- `BattleCharacter.hp` / `BattleCharacter.maxHp` - 当前血量

**MVP 计算逻辑：**
1. 存活单位优先
2. 击杀数最多
3. 击杀数相同则比较造成伤害

3. **金币奖励规则** - 连胜奖惩机制
4. **结算信息内容** - 击杀数、存活数、剩余血量、伤害统计
5. **失败视觉效果** - 明显区分（红色/灰色主题，💀 图标）
6. **连胜信息显示** - 显示当前连胜/连败 streak
7. **MVP 单位** - 显示本轮表现最好的单位
8. **敌方信息** - 显示敌方存活和造成伤害统计
