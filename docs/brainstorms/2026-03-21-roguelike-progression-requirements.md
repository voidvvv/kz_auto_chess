# Roguelike 难度递增模式

## Problem Frame

当前游戏内容过于单调，玩家选择关卡后只能重复挑战同一关卡，敌人难度固定无变化。缺乏长期挑战目标，游戏循环浅层化，难以维持玩家兴趣。

本功能旨在引入类似《杀戮尖塔》或《吸血鬼幸存者》的Roguelike递增模式，提供：
- 30个连续阶段的难度递增挑战
- 随机事件带来的策略选择
- Boss战作为里程碑挑战
- 元游戏系统提供长期目标

## Requirements

### R1. 游戏流程模式
点击「无尽模式」入口后，玩家经历连续的30个阶段战斗：
- 每阶段包含：战斗阶段 → 备战阶段 → 下一阶段
- 第1-9阶段：普通敌人
- 第10阶段：小Boss
- 第11-19阶段：普通敌人（难度提升）
- 第20阶段：小Boss
- 第21-29阶段：普通敌人（难度进一步提升）
- 第30阶段：最终Boss

### R2. 备战阶段界面
每阶段战斗结束后进入独立备战界面，玩家可以：
- 使用金币购买卡牌
- 升级已有卡牌（3张相同卡牌合并）
- 刷新商店（消耗金币）
- 点击「开始下一阶段」按钮继续

备战界面显示：
- 当前阶段数（如 "第 5 / 30 阶段"）
- 当前金币数量
- 玩家当前血量
- 商店卡牌列表
- 操作按钮（购买、升级、刷新、下一阶段）

### R3. 随机事件系统
每完成3个阶段后触发随机事件（第3、6、9、12、...、27阶段）：
- 事件触发时显示独立的事件界面
- 玩家从3个随机选项中选择1个
- 选项效果包括：增益效果、金币奖励、卡牌奖励、debuff但高回报等

事件示例：
- 「神秘商人」：获得3张随机T2卡牌，失去5金币
- 「训练场」：所有单位攻击力+10%，失去3血量
- 「宝箱」：获得10金币
- 「诅咒祭坛」：所有单位血量-20%，获得15金币

### R4. Boss阶段设计
- 第10/20阶段：小Boss（单个强力敌人）
- 第30阶段：最终Boss（单个超强敌人）

**Boss特殊技能**：
Boss拥有独特的技能机制，区别于普通敌人：
- 全屏AOE伤害
- 召唤小怪
- 自身强化（攻击力/血量提升）
- 特定区域效果

Boss战特点：
- 界面显示 "BOSS 战斗" 标识
- 战斗胜利后给予额外奖励（金币/稀有卡牌）
- 最终Boss胜利后显示通关画面

**Boss技能配置**：
在JSON配置中定义Boss的技能列表，例如：
```json
{
  "stage": 30,
  "isBoss": true,
  "bossType": "final_boss",
  "skills": ["aoe_damage", "summon_minions", "self_buff"]
}
```

### R5. 关卡配置系统（JSON驱动）
创建 `assets/roguelike_stage_config.json` 配置文件，每个阶段定义：

```json
{
  "stages": [
    {
      "stage": 1,
      "enemyCount": 3,
      "enemyMinTier": 1,
      "enemyMaxTier": 2,
      "battlefieldEffect": null
    },
    {
      "stage": 5,
      "enemyCount": 5,
      "enemyMinTier": 2,
      "enemyMaxTier": 3,
      "battlefieldEffect": {
        "type": "weather",
        "effect": "rain",
        "description": "雨天：所有单位移动速度-20%"
      }
    },
    {
      "stage": 10,
      "enemyCount": 1,
      "enemyMinTier": 4,
      "enemyMaxTier": 4,
      "isBoss": true,
      "bossType": "mini_boss_1"
    }
  ]
}
```

### R6. 战场效果系统
支持两种战场效果类型：

**天气系统**：
- 晴天（默认）：无特殊效果
- 雨天：所有单位移动速度-20%
- 雾天：攻击命中率-15%
- 风天：远程单位射程+30%

**区域效果**：
- 毒性区域：进入区域每秒扣血
- 治疗区域：进入区域每秒回血
- 加速区域：进入区域移动速度+50%

效果显示：
- 战场背景或图标显示当前生效效果
- 鼠标悬停显示效果描述

### R7. 元游戏渐进系统（成就系统）
首次游戏结束后，根据达成条件解锁成就：

**成就类型**（纯收集，不影响战斗平衡）：
- 「初次挑战」：完成第1阶段 → 解锁成就徽章
- 「初级冒险者」：到达第5阶段 → 解锁成就徽章
- 「Boss猎人」：击败第10阶段Boss → 解锁成就徽章
- 「生存专家」：到达第20阶段 → 解锁成就徽章
- 「通关者」：击败最终Boss → 解锁稀有成就徽章

**成就展示**：
- 主菜单新增「成就」页面
- 显示已解锁/未解锁的成就列表
- 成就包含：图标、名称、解锁条件、描述

成就进度保存在本地文件中，跨游戏会话持久化。

### R8. 游戏结束处理
- **失败**：显示「游戏结束」界面，显示到达阶段数、解锁的成就
- **通关**：显示「胜利」界面，显示通关星级、解锁的成就

两个选项：
- 「返回主菜单」
- 「再次挑战」（应用已解锁的永久升级）

## Success Criteria

- [ ] 玩家能够连续体验30个阶段的难度递增战斗
- [ ] 每3阶段正确触发随机事件，玩家能够选择并应用效果
- [ ] Boss阶段正确标识并提供额外挑战
- [ ] 备战界面正常工作，玩家能够购买/升级/刷新卡牌
- [ ] 修改JSON配置文件后，游戏行为相应变化（无需重新编译）
- [ ] 战场效果正确应用并视觉呈现
- [ ] 成就系统正确追踪并应用永久升级
- [ ] 失败/通关界面正确显示并提供重玩选项

## Scope Boundaries

### 包含
- 30阶段连续战斗流程
- 独立备战界面
- 随机事件系统（基础版3-5个事件）
- Boss战（第10、20、30阶段）
- JSON关卡配置
- 天气效果（2-3种）
- 区域效果（2-3种）
- 成就系统（5个基础成就）

### 不包含
- 多存档系统（仅单存档）
- 云同步成就
- 在线排行榜
- 复杂的天气动画（仅图标+文字说明）
- Boss特殊技能机制（第一阶段仅属性强化）

## Key Decisions

### 决策1：采用独立分支架构
**理由**：
- 保留现有5关卡系统作为「教学/练习模式」
- 完全独立的 `RoguelikeGameMode` 和 `RoguelikeScreen`
- 最小化对现有代码的影响
- 降低开发风险

**影响**：
- 需要创建新的 GameMode 实现
- 需要创建新的 Screen 实现
- 可以复用现有的 Manager（BattleManager、EconomyManager、CardManager）

### 决策2：事件固定间隔触发
**理由**：
- 每3阶段触发固定且可预期
- 玩家可以提前规划策略
- 简化实现和平衡

**触发阶段**：3, 6, 9, 12, 15, 18, 21, 24, 27（共9次事件）

### 决策3：Boss固定周期出现
**理由**：
- 每10阶段的Boss作为里程碑
- 玩家有明确的短期目标
- 第30阶段作为最终挑战

**Boss阶段**：10（小Boss）、20（小Boss）、30（最终Boss）

## Dependencies / Assumptions

**依赖现有系统**：
- `BattleManager` - 战斗管理
- `EconomyManager` - 经济系统
- `CardManager` - 卡牌管理
- `PlayerLifeBlackboard` - 玩家血量
- `GameEventSystem` - 事件系统

**假设**：
- 现有的卡池系统（SharedCardPool）适用于无尽模式
- 玩家血量跨阶段持久化（已实现）
- JSON配置文件在 assets 目录下可访问
