# 羁绊视觉反馈 (Synergy Visual Feedback)

## Problem Frame

玩家在自走棋游戏中无法看到当前羁绊状态，无法做出策略决策：
- 不知道已激活哪些羁绊
- 不知道离下一级还差多少
- 不知道添加某个角色能激活什么新羁绊

这导致策略性游戏变成"盲目购买"，失去了自走棋的核心乐趣。

## Requirements

### R1. 左侧羁绊面板
在屏幕左侧显示独立的固定羁绊面板，垂直列表布局。

### R2. 羁绊图标显示
每个羁绊使用图标 + 名称 + 等级/进度的形式显示：
- 图标：代表羁绊类型的图标（战士=剑，法师=魔法球等）
- 名称：羁绊中文名
- 等级/进度：如 `Lv.2` 或 `2/3`

### R3. 显示范围
面板显示以下羁绊：
- **已激活羁绊**（等级 ≥ 1）
- **即将激活羁绊**（等级 = 0 但上场数量 ≥ 1）
- 不显示完全未涉及的羁绊（减少视觉噪音）

### R4. 状态颜色区分
- **已激活**：金色边框/背景 + 图标发光
- **即将激活**：高亮颜色（如浅蓝）+ 进度数字
- **视觉反馈**：激活时有短暂闪烁动画

### R5. 进度信息
每个羁绊显示当前数量和下一级阈值：
- 已激活：`当前数量/下一级阈值`（如战士 Lv.2 显示 `4/6`）
- 即将激活：`当前数量/激活阈值`（如 `2/3` 表示差1个激活）

### R6. 悬停效果详情
鼠标悬停在羁绊上时，显示弹窗：
- 羁绊名称 + 当前等级
- 具体加成数值（如 "攻击力 +10%, 防御力 +20%"）
- 下一级加成预览（如果有下一级）

## Success Criteria

1. 玩家可以一眼看出当前激活了哪些羁绊
2. 玩家可以快速判断"再买一个法师就能激活 Mage(3)"
3. 悬停可以看到具体的数值加成
4. 面板不遮挡核心游戏区域

## Scope Boundaries

### In Scope
- 羁绊状态显示面板
- 图标 + 进度 + 悬停详情
- 颜色状态区分

### Out of Scope
- 羁绊效果数值调整（使用现有 SynergyManager 实现）
- 新增羁绊类型
- 战斗中的羁绊动画效果
- 移动端适配

## Key Decisions

1. **独立面板 vs 整合**：选择独立面板，信息更清晰
2. **显示范围**：只显示相关的羁绊（已激活或即将激活），避免8个羁绊全部显示造成混乱
3. **图标化 vs 纯文字**：图标化更直观，但需要设计/使用图标资源
4. **布局方式**：垂直列表，适合屏幕左侧空间

## Dependencies / Assumptions

- `SynergyManager` 已完整实现所有计算逻辑
- `SynergyType.getActivationThresholds()` 提供阈值数组
- `SynergyEffect` 提供12种加成类型
- 项目有可用的图标资源或使用文字/形状代替
