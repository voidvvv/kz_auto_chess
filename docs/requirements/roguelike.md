# 需求描述
目前游戏内容过于单调，在选择一个
关卡后，只能不断重复挑战同一关卡，然后敌人也是相同难度，毫无变化。我想要难度递增形的玩法。
例如：在玩家点击开始游戏后，会进入第一关，第一关会随机生成比较弱小的敌人，
打败后会给玩家金币反馈，玩家可以使用获取到的金币来在商店内购买新的卡牌，在备战完毕
后，会进入下一阶段，生成更强一些的敌人。然后每隔若干阶段，就会出现随机事件来进行选
择，会给予玩家不同的增益（或者debuff?）。此外，游戏还需要有一个最终阶段，即最终大b
oss，需要有一定的难度，需要在前期进行积累构筑队伍才能击败。需要注意，每个关卡难度
递增，但是关卡生成的敌人需要是随机（或者可配置）的，不能写死在代码里。

# 细节描述
- 每个阶段的敌人生成需要是随机的，可以通过配置文件（如JSON）来定义每个阶段可能出现的敌人类型和数量。
- 随机事件可以在每隔若干阶段（如每3关）触发一次，事件类型可以包括增益（如金币奖励、属性提升）和debuff。
- 在备战界面，需要可以看到下一场战斗敌人的阵型和属性，以便玩家进行针对性的准备。
- 最终Boss需要有独特的技能和较高的属性，玩家需要通过前期的积累和构筑来击败它。
- 需要在主界面新增一个「闯关模式」的按钮，玩家点击后进入这个新的游戏模式。
- 新的roguelike模式需要有一个专门的screen来控制，与当前的gamescreen分开，以便于管理不同的游戏流程和UI。

# 游戏大概模式
1. 玩家点击「闯关模式」进入游戏。
2. 进入第一关，随机生成弱小敌人。
3. 此时玩家进入第一次备战界面，可以购买卡牌、调整队伍。
4. 备战完毕，点击开始战斗，进行战斗，胜利后获取金币，失败后触发失败事件。
5. 上一场战斗胜利，则进入下一关，随机生成更强的敌人，然后玩家继续备战、战斗。备战阶段可以调整当前队伍，购买卡牌等。
6. 每隔若干关（如每3关）触发一次随机事件，玩家从3个选项中选择一个，获得增益或debuff。
7. 每场战斗结束后，玩家的阵容都会保留到下一场战斗，玩家可以根据敌人的阵容来进行调整。
8. 最终Boss关需要玩家积累足够的资源和构筑强力的队伍才能击败。

# 注意事项
这个需求可能非常复杂，需要涉及到多个系统的改动（敌人生成、商店系统、事件系统、UI等），建议分多个阶段实现，先实现基本的关卡递增和敌人生成，再逐步添加随机事件和最终Boss等内容。同时需要注意代码的可维护性和扩展性，避免过多的硬编码，尽量通过配置文件来定义游戏内容。
